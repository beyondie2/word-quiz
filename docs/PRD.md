# 영단어 외우기 앱
## 1. 프로젝트 개요
### 1.1 프로젝트명
영단어 외우기 앱
### 1.2 프로젝트 목적
사용자가 주어진 단어를 보고, 그 의미를 맞추게 하고, 관리자가 학습 과정을 관리할 수 있는 간단한 풀스택 웹 앱
### 1.3 개발 범위
- 단어 맞추기 화면(회원 확인 기능, 책 선택 기능, 장 선택 기능, 단어 맞추기 기능)
- 수행 확인 화면(틀린 답, 책, 단원, 오답, 학습모드, 한국어대답방법, 라운드, 단원전체 재학습 회차, 일시 등을 확인)
- 데이터를 생성/조회/수정/삭제할 수 있는 기능
## 2. 기술 스택
- 프런트엔드: HTML, CSS, 리액트, 자바스크립트
- 백엔드: Node.js, Express
- 데이터베이스: PostgreSQL
## 3. 기본 사항
- 프런트엔드와 백엔드를 따로 개발
- 기본적인 웹 기술만 사용
- 결제 기능은 제외
- 영어를 보고 한국어로 맞추고, 한국어를 보고 영어를 맞춤

## 4. 프런트엔드 UI 명세

### 4.1 단어 맞추기 화면

#### 4.1.1 화면 구성

| 영역 | 구성 요소 | 설명 |
|------|----------|------|
| 헤더 | 로고, 탭 메뉴 | 앱 타이틀과 화면 전환 탭 |
| 설정 바 | 사용자 인증, 설정 버튼 및 옵션, 단원 선택 | 사용자 확인 및 학습 조건 설정 |
| 문제 영역 | 출제 단어, 한국어 힌트 | 맞춰야 할 단어 표시 |
| 입력 영역 | 텍스트 입력 필드 | 정답 입력 |
| 하단 버튼 | 액션 버튼 | 학습 진행 제어 |

#### 4.1.2 헤더 영역

| 요소 | 타입 | 설명 |
|------|------|------|
| WORD TEST | 텍스트(로고) | 앱 타이틀, 좌측 배치 |
| 단어 맞추기 | 탭 버튼 | 현재 화면 (활성 상태 시 초록색 테두리) |
| 수행 확인 | 탭 버튼 | 수행 확인 화면으로 이동 |

#### 4.1.3 설정 바 영역

| 요소 | 타입 | 옵션 | 설명 |
|------|------|------|------|
| 이름 | 텍스트 입력 | - | 사용자 본인 인증을 위한 이름 입력 |
| 단어장선택 | 버튼 | - | 클릭 시 단어장 목록 표시 (모달/드롭다운) |
| 연습방법 | 라디오 버튼 | 한국어 / 영어 | 문제로 출제되는 언어 선택 |
| 한국어답 | 라디오 버튼 | 하나만 / 전부다 | 한국어 정답 입력 방식 선택 |
| 부분단원 | 버튼 | - | 클릭 시 단원 세부 선택 (모달/드롭다운) |

**이름 입력 필드 상세:**
- 플레이스홀더: "이름"
- 사용자 본인 인증 용도 (학습 기록 저장 시 사용자 식별에 사용)
- 학습 시작 전 필수 입력 권장

**연습방법 옵션 상세:**
- `한국어`: 한국어 단어를 보고 영어 뜻을 맞춤
- `영어`: 영어 단어를 보고 한국어 뜻을 맞춤

**한국어답 옵션 상세:**
- `하나만`: 여러 개의 한국어 뜻 중 하나만 입력해도 정답 처리
- `전부다`: 모든 한국어 뜻을 전부 입력해야 정답 처리

#### 4.1.4 문제 영역

| 요소 | 타입 | 설명 |
|------|------|------|
| 출제 단어 | 텍스트 | 화면 중앙에 크게 표시, 맞춰야 할 단어, 한국어 또는 영어로 표시 |

**표시 규칙:**
- 연습방법이 `영어`인 경우: 영어 단어 표시
- 연습방법이 `한국어`인 경우: 한국어 단어 표시

#### 4.1.5 입력 영역

| 요소 | 타입 | 설명 |
|------|------|------|
| 정답 입력 필드 | 텍스트 입력 | 사용자가 정답을 입력하는 필드 |

**입력 필드 상세:**
- 플레이스홀더: 연습방법에 따라 동적 변경
  - `영어` 선택 시: "한국어_의미"
  - `한국어` 선택 시: "영어_의미"
- 입력 후 Enter 키 또는 제출 버튼으로 정답 확인

#### 4.1.6 하단 버튼 영역

| 요소 | 타입 | 설명 |
|------|------|------|
| 틀린 것만 다시하기 | 버튼 | 이전에 틀린 문제들만 다시 출제 |

#### 4.1.7 사용자 흐름 (User Flow)

```
1. 화면 진입
   ↓
2. [이름] 입력 → 사용자 본인 인증
   ↓
3. [단어장선택] 버튼 클릭 → 단어장 목록에서 선택
   ↓
4. (선택) [부분단원] 버튼 클릭 → 특정 단원/범위 선택
   ↓
5. [연습방법] 선택 (한국어/영어)
   ↓
6. [한국어답] 옵션 선택 (하나만/전부다)
   ↓
7. 출제된 단어 확인
   ↓
8. 정답 입력 → Enter 또는 제출
   ↓
9. 정답/오답 피드백 표시
   ↓
10. 다음 문제로 이동 (반복)
   ↓
11. (선택) [틀린 것만 다시하기] 클릭 → 오답 문제 재출제
```

#### 4.1.8 상태 및 피드백

| 상태 | 시각적 피드백 |
|------|--------------|
| 정답 | 입력 필드 테두리 초록색, 정답 메시지 표시 |
| 오답 | 입력 필드 테두리 빨간색, 정답 표시 |
| 미입력 | 기본 상태 유지 |
| 이름 미입력 | 학습 시작 시 경고 메시지 표시 |

#### 4.1.9 디자인 가이드

**색상:**
- 기본 테마 색상: 초록색 (#4CAF50 또는 유사)
- 활성 탭/버튼 테두리: 초록색
- 오답 표시: 빨간색

**레이아웃:**
- 전체 컨테이너: 가운데 정렬, 최대 너비 제한
- 설정 바: 이름 → 단어장선택 → 연습방법 → 한국어답 → 부분단원 순서로 배치
- 각 영역 간 적절한 여백
- 모바일 반응형 지원

### 4.2 문법 익히기 화면

#### 4.2.1 화면 개요

문법 익히기 화면은 사용자가 영어 문법을 학습할 수 있는 화면입니다. 문제 전체에 대한 지시문을 보고, 각 문항에 맞는 정답을 입력하는 방식으로 학습합니다.

#### 4.2.2 화면 구성

| 영역 | 구성 요소 | 설명 |
|------|----------|------|
| 헤더 | 로고, 탭 메뉴 | 앱 타이틀과 화면 전환 탭 |
| 설정 바 | 사용자 정보, 분류 선택 버튼 | 사용자 확인 및 문법 분류 선택 |
| 지시문 영역 | 지시문 텍스트 | 문제 전체에 대한 안내 |
| 문항 영역 | 문항 내용 | 개별 문항 표시 |
| 입력 영역 | 정답 입력 필드 | 사용자 정답 입력 |

#### 4.2.3 헤더 영역

| 요소 | 타입 | 설명 |
|------|------|------|
| WORD TEST | 텍스트(로고) | 앱 타이틀, 좌측 배치 |
| 단어 맞추기 | 탭 버튼 | 단어 맞추기 화면으로 이동 |
| 문법 익히기 | 탭 버튼 | 현재 화면 (활성 상태 시 강조 표시) |
| 수행 확인 | 탭 버튼 | 수행 확인 화면으로 이동 |
| 관리자 | 탭 버튼 | 관리자 화면으로 이동 (관리자만 표시) |

#### 4.2.4 설정 바 영역

| 요소 | 타입 | 설명 |
|------|------|------|
| 사용자 이름 | 텍스트 | 로그인한 사용자 이름 표시 |
| 분류1 | 드롭다운 버튼 | 문법 대분류 선택 (예: 시제, 품사, 문장구조 등) |
| 분류2 | 드롭다운 버튼 | 문법 중분류 선택 (예: 현재시제, 과거시제 등) |
| 분류3 | 드롭다운 버튼 | 문법 소분류 선택 (예: 현재완료, 현재진행 등) |

**분류 체계 예시:**
- 분류1 (대분류): 시제, 조동사, 가정법, 수동태, 관계사, 접속사 등
- 분류2 (중분류): 현재, 과거, 미래, 완료 등
- 분류3 (소분류): 긍정문, 부정문, 의문문, 특수구문 등

#### 4.2.5 지시문 영역

| 요소 | 타입 | 설명 |
|------|------|------|
| 지시문 박스 | 컨테이너 | 문제 세트 전체에 대한 지시사항 표시 |

**지시문 예시:**
- "다음 문장의 빈칸에 알맞은 단어를 쓰시오."
- "밑줄 친 부분 중 어법상 틀린 것을 고르시오."
- "주어진 단어를 올바른 형태로 바꿔 쓰시오."
- "다음 문장을 영작하시오."

**디자인:**
- 왼쪽에 강조 테두리 (accent color)
- 배경색으로 다른 영역과 구분

#### 4.2.6 문항 영역

| 요소 | 타입 | 설명 |
|------|------|------|
| 문항 번호 | 텍스트 | 현재 문항 번호 표시 |
| 문항 내용 | 텍스트/HTML | 문제 본문 표시 (빈칸, 밑줄 등 포함 가능) |
| 보기 | 선택지 목록 | 객관식인 경우 선택지 표시 (선택적) |

**문항 유형:**
1. **빈칸 채우기**: 문장 내 빈칸(______)에 알맞은 단어/구 입력
2. **어형 변환**: 주어진 단어를 문맥에 맞게 변형
3. **오류 수정**: 문장에서 틀린 부분 찾아 수정
4. **영작**: 한글 문장을 영어로 작성
5. **객관식**: 여러 선택지 중 정답 선택

#### 4.2.7 입력 영역

| 요소 | 타입 | 설명 |
|------|------|------|
| 정답 입력 필드 | 텍스트 입력 | 사용자가 정답을 입력하는 필드 |

**입력 필드 상세:**
- 플레이스홀더: "정답 입력"
- Enter 키 또는 제출 버튼으로 정답 확인
- 정답/오답에 따라 테두리 색상 변경

#### 4.2.8 사용자 흐름 (User Flow)

```
1. 화면 진입 (로그인 필요)
   ↓
2. [분류1] 선택 → 대분류 선택 (시제, 품사 등)
   ↓
3. [분류2] 선택 → 중분류 선택 (현재, 과거 등)
   ↓
4. [분류3] 선택 → 소분류 선택 (긍정문, 부정문 등)
   ↓
5. 지시문 확인
   ↓
6. 문항 내용 확인
   ↓
7. 정답 입력 → Enter 또는 제출
   ↓
8. 정답/오답 피드백 표시
   ↓
9. 다음 문항으로 이동 (반복)
   ↓
10. 학습 완료 시 결과 요약 표시
```

#### 4.2.9 상태 및 피드백

| 상태 | 시각적 피드백 |
|------|--------------|
| 정답 | 입력 필드 테두리 초록색, 정답 메시지 표시 |
| 오답 | 입력 필드 테두리 빨간색, 정답 및 해설 표시 |
| 미입력 | 기본 상태 유지 |
| 로그인 필요 | 로그인 안내 메시지 표시 |

#### 4.2.10 디자인 가이드

**색상:**
- 지시문 영역 강조: #5BA3B5 (좌측 테두리)
- 문항 영역 배경: #0D2535
- 정답 표시: #4ADE80 (초록색)
- 오답 표시: #F87171 (빨간색)

**레이아웃:**
- 설정 바: 사용자 이름 → 분류1 → 분류2 → 분류3 순서로 배치
- 지시문 영역: 눈에 띄게 구분되는 박스 스타일
- 문항 영역: 충분한 여백과 가독성 있는 폰트
- 입력 영역: 넓은 입력 필드
- 모바일 반응형 지원

### 4.3 수행 확인 화면

#### 4.3.1 화면 구성

| 영역 | 구성 요소 | 설명 |
|------|----------|------|
| 헤더 | 로고, 탭 메뉴 | 앱 타이틀과 화면 전환 탭 |
| 필터 영역 | 사용자 선택, 일시 선택 | 조회 조건 설정 |
| 콘텐츠 영역 | 수행 내용 목록 | 학습 수행 결과 표시 |

#### 4.3.2 헤더 영역

| 요소 | 타입 | 설명 |
|------|------|------|
| WORD TEST | 텍스트(로고) | 앱 타이틀, 좌측 배치 |
| 단어 맞추기 | 탭 버튼 | 단어 맞추기 화면으로 이동 |
| 문법 익히기 | 탭 버튼 | 문법 익히기 화면으로 이동 |
| 수행 확인 | 탭 버튼 | 수행내용과 정도를 확인하는 화면으로 이동 (활성 상태 시 초록색 테두리) |

#### 4.3.3 필터 영역

| 요소 | 타입 | 설명 |
|------|------|------|
| 사용자 이름 | 드롭다운/버튼 | 조회할 사용자 선택 |
| 과제한 날짜 | 날짜 선택기 | 조회할 날짜 또는 날짜 범위 선택 |

**필터 동작:**
- 사용자 이름 선택 시: 해당 사용자의 수행 내용만 표시
- 과제한 날짜 선택 시: 해당 날짜에 수행된 내용만 표시
- 두 필터 조합 가능: 특정 사용자의 특정 날짜 수행 내용 조회

#### 4.3.4 콘텐츠 영역 (수행 내용)

| 요소 | 타입 | 설명 |
|------|------|------|
| 수행 내용 박스 | 컨테이너 | 학습 수행 결과를 표시하는 영역 |

**수행 내용에 포함될 정보:**
- 학습한 단어장 이름
- 학습한 단원/범위
- 학습 일시
- 총 문제 수
- 정답 수 / 오답 수
- 정답률
- 틀린 단어 목록

**틀린 단어 목록 표시 형식:**

| 항목 | 설명 |
|------|------|
| 영어 단어 | 틀린 문제의 영어 단어 |
| 한국어 뜻 | 해당 단어의 정답 (한국어 뜻) |
| 사용자 입력 | 사용자가 입력한 오답 |

#### 4.3.5 사용자 흐름 (User Flow)

```
1. 화면 진입
   ↓
2. (선택) [사용자 이름] 선택 → 특정 사용자 필터링
   ↓
3. (선택) [과제한 날짜] 선택 → 특정 날짜 필터링
   ↓
4. 수행 내용 목록 확인
   ↓
5. 특정 수행 기록 클릭 → 상세 내용 확인 (틀린 단어 목록 등)
```

#### 4.3.6 데이터 표시 상태

| 상태 | 표시 내용 |
|------|----------|
| 데이터 있음 | 수행 내용 목록 표시 |
| 데이터 없음 | "수행 기록이 없습니다" 메시지 표시 |
| 로딩 중 | 로딩 인디케이터 표시 |

#### 4.3.7 디자인 가이드

**색상:**
- 기본 테마 색상: 초록색 (#4CAF50 또는 유사)
- 활성 탭 테두리: 초록색
- 정답률 표시: 높음(초록), 중간(노랑), 낮음(빨강)

**레이아웃:**
- 필터 영역: 상단 좌측 정렬
- 콘텐츠 영역: 필터 아래 전체 너비 사용
- 수행 내용 박스: 둥근 모서리, 테두리 스타일
- 모바일 반응형 지원

| created_at | TIMESTAMP | 학습 일시 | DEFAULT NOW() |

### 5.2 데이터 스키마를 위한 사용자 흐름

```
① 사용자가 자기 이름을 입력하면
   → 책 선택 select가 활성화됨
   → username input은 보이지 않게 됨
   ↓
② 사용자는 "단어장선택" select에서 단어장을 선택함
   ↓
③ 사용자는 연습방법 radio group에서 선택
   - 한국어를 보고 영어로 답할지
   - 영어를 보고 한국어로 답할지
   ↓
④ 사용자는 한국어답 radio group에서 선택
   - 한국어 단어 하나만 써서 답할지
   - 여러 표현 전부를 써서 답할지
   ↓
⑤ 사용자는 부분단원 select에서 단원을 선택
   → 앱은 한국어 또는 영어로 된 단어를 보여줌
   ↓
⑥ 사용자는 주어진 단어를 보고 입력상자에 정답을 입력함
   ↓
⑦ 정답 확인
   - 맞으면: 다음 단어로 넘어감
   - 틀리면: modal로 정답을 보여줌
   ↓
⑧ 정해진 개수의 단어를 다 했으면
   → "틀린 것만 다시하기" 버튼을 보여줌
   ↓
⑨ 사용자가 "틀린 것만 다시하기" 버튼을 클릭하면
   → 틀린 단어의 목록 중 첫번째 것을 보여줌 (반복)
   ↓
⑩ "틀린 것만 다시하기"를 누를 때마다
   → round 횟수를 증가시켜 round 횟수를 알려줌
   ↓
⑪ 틀린 것이 없으면
   → 성공 메시지를 보여주고 return 함
```

### 5.3 API 설계

#### 5.3.1 사용자 인증 API

**POST /api/users/verify**

사용자 이름을 확인하고 책 목록을 반환합니다.

| 항목 | 내용 |
|------|------|
| 요청 본문 | `{ "username": "사용자이름" }` |
| 성공 응답 | `{ "success": true, "userId": 1, "books": ["책1", "책2", ...] }` |
| 실패 응답 | `{ "success": false, "message": "등록되지 않은 사용자입니다" }` |

**동작:**
- 데이터베이스 Users 테이블에서 이름 목록과 대조
- 이름이 있으면 Books 테이블의 고유한 book_name 목록을 반환

---

#### 5.3.2 단원 목록 API

**GET /api/books/:bookName/units**

선택한 책의 단원 목록을 반환합니다.

| 항목 | 내용 |
|------|------|
| URL 파라미터 | `bookName` - 책 이름 |
| 성공 응답 | `{ "units": ["Unit 1", "Unit 2", ...] }` |

**동작:**
- Books 테이블에서 해당 책의 고유한 unit 값들을 반환

---

#### 5.3.3 단어 목록 API

**GET /api/books/:bookName/units/:unit/words**

선택한 책과 단원의 단어 목록을 반환합니다.

| 항목 | 내용 |
|------|------|
| URL 파라미터 | `bookName` - 책 이름, `unit` - 단원 |
| 성공 응답 | `{ "words": [{ "id": 1, "english": "apple", "korean": "사과", "example": "I eat an apple" }, ...] }` |

**동작:**
- Books 테이블에서 해당 책, 단원의 모든 단어를 반환
- 첫 번째 단어부터 순서대로 학습 시작

---

#### 5.3.4 정답 확인 API

**POST /api/quiz/check**

사용자의 답을 확인하고 결과를 반환합니다.

| 항목 | 내용 |
|------|------|
| 요청 본문 | `{ "userId": 1, "wordId": 1, "userAnswer": "사과", "practiceMode": "english", "koreanAnswerType": "one", "bookName": "책이름", "unit": "Unit 1", "round": 1, "unitReviewCount": 0 }` |
| 정답 응답 | `{ "correct": true, "nextWord": { ... } }` |
| 오답 응답 | `{ "correct": false, "correctAnswer": "사과, 능금", "nextWord": { ... } }` |

**동작:**
- practiceMode가 'english'면 사용자가 한국어를 입력함 → korean 필드와 비교
- practiceMode가 'korean'면 사용자가 영어를 입력함 → english 필드와 비교
- koreanAnswerType이 'one'면 쉼표로 구분된 한국어 중 하나만 맞아도 정답
- koreanAnswerType이 'all'면 모든 한국어 표현을 입력해야 정답
- UserProgress 테이블에 학습 기록 저장 (맞음/틀림 모두 기록)

---

#### 5.3.5 학습 기록 저장 API

**POST /api/progress**

사용자의 학습 수행 정보를 기록합니다.

| 항목 | 내용 |
|------|------|
| 요청 본문 | `{ "userId": 1, "bookName": "책이름", "unit": "Unit 1", "english": "apple", "korean": "사과", "wrongAnswer": "오렌지", "practiceMode": "english", "koreanAnswerType": "one", "round": 1, "unitReviewCount": 0, "isCorrect": false }` |
| 성공 응답 | `{ "success": true, "progressId": 1 }` |

**동작:**
- UserProgress 테이블에 모든 학습 정보 저장

---

#### 5.3.6 틀린 단어 목록 API

**GET /api/progress/:userId/wrong-words**

사용자의 틀린 단어 목록을 반환합니다.

| 항목 | 내용 |
|------|------|
| URL 파라미터 | `userId` - 사용자 ID |
| 쿼리 파라미터 | `bookName` - 책 이름, `unit` - 단원, `round` - 현재 라운드 |
| 성공 응답 | `{ "wrongWords": [{ "id": 1, "english": "apple", "korean": "사과", "wrongAnswer": "오렌지" }, ...] }` |
| 빈 목록 응답 | `{ "wrongWords": [], "message": "모든 단어를 맞추셨습니다!" }` |

**동작:**
- "틀린 것만 다시하기" 버튼 클릭 시 호출
- 해당 사용자의 현재 라운드에서 틀린 단어 목록 반환
- 틀린 단어가 없으면 성공 메시지와 함께 빈 배열 반환

---

#### 5.3.7 라운드 증가 API

**POST /api/progress/:userId/next-round**

"틀린 것만 다시하기" 클릭 시 라운드를 증가시킵니다.

| 항목 | 내용 |
|------|------|
| URL 파라미터 | `userId` - 사용자 ID |
| 요청 본문 | `{ "bookName": "책이름", "unit": "Unit 1" }` |
| 성공 응답 | `{ "success": true, "newRound": 2 }` |

**동작:**
- 현재 라운드 번호를 1 증가시킴
- 새로운 라운드 번호 반환

---

#### 5.3.8 수행 기록 조회 API

**GET /api/progress**

수행 확인 화면에서 사용하는 학습 기록 조회 API입니다.

| 항목 | 내용 |
|------|------|
| 쿼리 파라미터 | `userId` - 사용자 ID (선택), `date` - 날짜 (선택, YYYY-MM-DD 형식) |
| 성공 응답 | `{ "records": [{ "id": 1, "bookName": "책이름", "unit": "Unit 1", "totalWords": 20, "correctCount": 15, "wrongCount": 5, "accuracy": 75, "wrongWords": [...], "createdAt": "2026-01-12T10:30:00Z" }, ...] }` |

**동작:**
- 필터 조건에 맞는 학습 기록 반환
- 정답률, 총 문제 수, 정답/오답 수 계산

---

#### 5.3.9 사용자 목록 API

**GET /api/users**

수행 확인 화면의 사용자 필터용 목록을 반환합니다.

| 항목 | 내용 |
|------|------|
| 성공 응답 | `{ "users": [{ "id": 1, "username": "홍길동" }, ...] }` |

### 5.4 API 흐름 요약

```
[사용자 이름 입력 + Enter]
    ↓
POST /api/users/verify
    → 성공: 책 목록 반환, 책 select 활성화
    → 실패: 에러 메시지 표시
    ↓
[단어장 선택]
    ↓
GET /api/books/:bookName/units
    → 단원 목록 반환
    ↓
[연습방법 선택] - 클라이언트에서 처리
    ↓
[한국어답 선택] - 클라이언트에서 처리
    ↓
[단원 선택]
    ↓
GET /api/books/:bookName/units/:unit/words
    → 단어 목록 반환, 첫 단어 표시
    ↓
[정답 입력]
    ↓
POST /api/quiz/check
    → 정답: 다음 단어 표시
    → 오답: modal로 정답 표시, 다음 단어 표시
    ↓

    → "틀린 것만 다시하기" 버튼 표시
    ↓
[틀린 것만 다시하기 클릭]
    ↓
POST /api/progress/:userId/next-round
    → 라운드 증가
    ↓
GET /api/progress/:userId/wrong-words
    → 틀린 단어 있음: 첫 번째 틀린 단어 표시

```
